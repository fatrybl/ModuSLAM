Tests of get_element() method overloads for any implementation of DataReader(ABC).

Any DataReader should work in 2 regimes: Stream and TimeLimit,
get_element() methods should be tested for both regimes.

 == Attention ==
    overloads which ignore time regimes:
        get_element(sensor: Sensor, timestamp: int)
        get_element(element: Element)
    They seek for the element with the given sensor_name & timestamp / element in the whole dataset.

1. get_element () -> Element | None:
    1.1 no args, a dataset is valid - > Element.
    1.2 no args, a dataset is not valid -> None.
    1.3 ...

2. get_elements (sensor) -> Element | None:
    2.1 sensor is initialized -> Element
    2.2 sensor is not initialized -> None
    2.3 ...

3. get_element (sensor, timestamp) -> Element:
    3.1 sensor and timestamp are valid -> Element
    3.2 sensor is not initialized, time is valid -> ItemNotExistsError
    3.3 sensor is initialized, time is not valid -> ItemNotExistsError
    3.4 sensor is not initialized, time is not valid -> ItemNotExistsError
    3.5 ...

4. get_element (element: Element) -> Element:
    4.1 element is valid: exists in dataset -> Element
    4.2 element is not valid -> ItemNotExistsError
    4.3 ...
